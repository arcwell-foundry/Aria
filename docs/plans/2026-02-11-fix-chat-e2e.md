# Fix Chat System End-to-End Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Fix the 3 real gaps in the chat system: messages not persisted to DB, streaming endpoint missing Digital Twin context, and chat response missing ARIA's standard response envelope (rich_content, ui_commands, suggestions).

**Architecture:** The chat endpoint and frontend already match (`POST /api/v1/chat` and `/chat/stream`). The real gaps are: (1) messages only live in RAM (WorkingMemory) and are lost on restart — the `messages` table exists but nothing writes to it; (2) the streaming endpoint skips `style_guidelines` and `priming_context` that the non-streaming path includes; (3) the response envelope lacks `rich_content`, `ui_commands`, and `suggestions` fields that ARIA needs to control the UI.

**Tech Stack:** Python/FastAPI, Supabase (PostgreSQL), TypeScript/React, SSE streaming

| 4 | Agents not wired to chat | LOW (deferred — skill system handles this) |

Agent wiring is deferred: the skill system already routes tasks to specialized capabilities. Connecting the 6 named agents is a separate architectural decision.

```python
async def save_message(
    self,
    conversation_id: str,
    role: str,
    content: str,
    metadata: dict[str, Any] | None = None,
) -> str:
    """Persist a message to the messages table.

    Args:
        conversation_id: The conversation this message belongs to.
        role: Message role ('user', 'assistant', 'system').
        content: Message text content.
        metadata: Optional metadata (skill execution, citations, etc.).

    Returns:
        The UUID of the saved message.
    """
    import uuid as _uuid

    message_id = str(_uuid.uuid4())
    self.db.table("messages").insert(
        {
            "id": message_id,
            "conversation_id": conversation_id,
            "role": role,
            "content": content,
            "metadata": metadata or {},
        }
    ).execute()

    return message_id
```

```python
async def get_conversation_messages(
    self,
    user_id: str,
    conversation_id: str,
) -> list[ConversationMessage]:
    """Get all messages for a conversation.

    Args:
        user_id: The user's ID.
        conversation_id: The conversation ID.

    Returns:
        List of ConversationMessage objects in chronological order.

    Raises:
        NotFoundError: If conversation doesn't belong to user.
    """
    # Verify ownership
    conv = (
        self.db.table("conversations")
        .select("id")
        .eq("user_id", user_id)
        .eq("id", conversation_id)
        .single()
        .execute()
    )

    if not conv.data:
        raise NotFoundError(resource="Conversation", resource_id=conversation_id)

    # Read from messages table
    result = (
        self.db.table("messages")
        .select("*")
        .eq("conversation_id", conversation_id)
        .order("created_at", desc=False)
        .execute()
    )

    if not result.data:
        return []

    return [ConversationMessage.from_dict(row) for row in result.data]
```
